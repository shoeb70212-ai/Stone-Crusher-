# Mobile Layout Modernization Plan

## Objective
Transform the CrushTrack mobile experience to feel like a modern native app — with polished navigation, accessible layouts, improved information hierarchy, and consistent design patterns across all screens.

## Scope
All mobile views (`md:hidden` / `@media (max-width: 640px)`) across the app. Desktop layouts remain largely untouched.

---

## Phase 1: Navigation & Shell Improvements

### 1.1 Bottom Tab Bar Redesign
**Files:** `src/components/Sidebar.tsx`, `src/index.css`

**Current Issues:**
- Tab bar is plain white with basic icons
- No active state animation beyond a tiny top indicator
- "More" button opens a full grid drawer that feels disconnected

**Plan:**
- Add a subtle frosted-glass effect (`backdrop-blur-xl bg-white/80 dark:bg-zinc-950/80`)
- Increase icon size to `w-6 h-6` with filled variants for active state
- Replace top-line indicator with a soft pill background behind the active tab
- Add a gentle scale animation on tab press (`active:scale-90`)
- Center the active tab label with slightly larger font
- Add haptic feedback on tab switch via `useHapticFeedback`

**Implementation:**
```tsx
// In Sidebar.tsx bottom nav section
// Replace the current active indicator with:
<div className="absolute inset-x-2 top-1 bottom-1 bg-primary-100 dark:bg-primary-500/20 rounded-2xl -z-10 transition-all duration-200" />
```

### 1.2 Floating Action Button (FAB) Polish
**Files:** `src/components/Layout.tsx`, `src/index.css`

**Plan:**
- Add a subtle shadow elevation (`shadow-2xl shadow-primary-500/30`)
- Add a press animation with spring physics
- Consider adding a label that appears on long-press for accessibility
- Ensure FAB doesn't overlap bottom sheet handles

### 1.3 Header Improvements
**Files:** `src/components/Header.tsx`

**Plan:**
- Add a subtle bottom border that fades on scroll (can be CSS-only with `position: sticky`)
- Make the mobile header slightly taller (`h-14`) for better touch targets
- Add a back button on sub-pages (when not on dashboard)
- Consider collapsing the header on scroll down to maximize content area

## Phase 3: List Views Redesign (Dispatch, Invoices, Customers, etc.)

### 3.1 Card-Based List Items
**Files:** `src/pages/Dispatch.tsx`, `src/pages/Invoices.tsx`, `src/pages/Customers.tsx`, `src/pages/Vehicles.tsx`, `src/pages/Employees.tsx`

**Current Issues:**
- Cards have too much internal padding, wasting vertical space
- Action buttons are small and cramped
- Status badges are inconsistent sizes
- Information hierarchy is flat — everything looks equally important

**Plan:**
- **Compact Card Design:**
  - Reduce outer padding to `p-3`
  - Use a 2-row layout: primary info top, metadata bottom
  - Status badge becomes a small dot + text on the right
  - Amount is large and bold, right-aligned
  
- **Swipe Actions:**
  - Implement swipe-left-to-reveal actions (Edit, Print, Share, Delete)
  - Use `react-swipeable` or native CSS touch events
  - Show action icons with color backgrounds
  
- **Staggered Entry Animation:**
  - Cards animate in with a slight delay between each
  - Use the existing `stagger-animation` CSS class

- **Empty States:**
  - Use the existing `EmptyState` component consistently
  - Add an illustration or icon with a clear CTA button

### 3.2 Filter & Search UX
**Files:** `src/pages/Dispatch.tsx`, `src/pages/Invoices.tsx`

**Plan:**
- Convert filter button into a persistent search bar at the top
- Show active filters as removable chips below the search bar
- Use a bottom sheet for advanced filters (already partially implemented)
- Add "Recent Searches" and "Quick Filters" suggestions

## Phase 4: Forms & Modals Modernization

### 4.1 Bottom Sheet Forms
**Files:** `src/components/ui/MobileModal.tsx`, `src/components/forms/CreateSlipForm.tsx`

**Current Issues:**
- Forms feel cramped with too many fields visible at once
- No step-by-step flow for complex operations
- Input fields have inconsistent heights

**Plan:**
- **Step-by-Step Slip Creation:**
  - Step 1: Vehicle & Driver (vehicle no, driver name, delivery mode)
  - Step 2: Material & Measurement (material type, dimensions/weight)
  - Step 3: Pricing & Customer (rate, customer, payment status)
  - Step 4: Review & Submit (summary card with all details)
  
- **Input Field Standardization:**
  - All inputs use `h-12` minimum height for touch
  - Labels float above inputs (not inline)
  - Use `type="tel"` for numeric fields to show numpad
  - Add input masks for vehicle numbers, phone numbers

- **Form Progress Indicator:**
  - Show a dot progress bar at the top of the modal
  - Allow tapping dots to jump between steps (if valid)

### 4.2 Confirmation & Action Sheets
**Files:** `src/components/ui/MobilePrimitives.tsx`

**Plan:**
- Use the existing `MobileActionSheet` but improve styling
- Add danger zone styling for destructive actions (red background, warning icon)
- Add haptic feedback before showing destructive confirmations
- Use iOS-style action sheets with cancel button always at bottom

## Phase 5: Accessibility & Polish

### 5.1 Touch Targets
**Files:** `src/index.css` (mobile utilities)

**Plan:**
- Ensure ALL interactive elements are minimum 44×44dp
- Add `touch-manipulation` to prevent double-tap zoom
- Increase button padding globally for mobile
- Add `active:` states with visible feedback

### 5.2 Typography & Spacing
**Files:** `src/index.css`

**Plan:**
- Use a slightly larger base font size on mobile (`15px` instead of `14px`)
- Increase line height for readability (`leading-relaxed`)
- Use consistent spacing scale (4px base unit)
- Ensure text contrast meets WCAG AA (4.5:1 minimum)

### 5.3 Dark Mode Consistency
**Files:** All page components

**Plan:**
- Audit all cards, borders, and backgrounds for dark mode
- Use `dark:` variants consistently
- Ensure shadows are visible in dark mode (use colored shadows)
- Test all status badges in both modes

### 5.4 Loading & Empty States
**Files:** `src/components/ui/Skeleton.tsx`, `src/components/ui/EmptyState.tsx`

**Plan:**
- Use skeleton screens instead of spinners for list loading
- Add shimmer animation to skeletons
- Empty states should have a clear icon, message, and primary action button

---

## Phase 6: Performance Optimizations

### 6.1 List Virtualization
**Files:** `src/pages/Dispatch.tsx`, `src/pages/Invoices.tsx`

**Plan:**
- Implement virtual scrolling for lists > 50 items
- Use `react-window` or CSS-only `content-visibility: auto`
- Add infinite scroll with skeleton placeholders

### 6.2 Image & Asset Optimization
**Files:** `src/lib/camera.ts`, `src/index.css`

**Plan:**
- Compress camera captures before storing
- Use WebP format where supported
- Lazy load images below the fold

### 6.3 Animation Performance
**Files:** `src/index.css`

**Plan:**
- Use `transform` and `opacity` only for animations (GPU-accelerated)
- Add `will-change` hints for frequently animated elements
- Respect `prefers-reduced-motion`
