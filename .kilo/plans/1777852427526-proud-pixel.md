validat# CrushTrack ERP — Comprehensive Codebase Audit Plan

## Objective

Perform a comprehensive audit of the entire CrushTrack ERP codebase and all associated documentation to identify logic gaps, architectural flaws, bugs, and optimization opportunities. Prioritize the mobile platform experience (Capacitor 8 Android build), analyzing mobile-specific /skimplementations for performance bottlenecks, UX inconsistencies, responsiveness issues, and platform-specific bugs.

## Deliverable

A detailed `AUDIT_REPORT.md` categorized by severity (Critical / High / Medium / Low) with actionable, file-level recommendations for improvement.

---

## Phase 1: Mobile-First Deep Dive (Priority)

### 1.1 Capacitor Integration & Native Bridge

- **Files:** `src/lib/capacitor.ts`, `src/lib/*.ts` (native wrappers)
- **Check:**
  - All native plugin imports are correctly lazy-loaded
  - `isNative()` guards are present and consistent
  - Plugin error handling is robust (no unhandled rejections)
  - Capacitor config (`capacitor.config.ts`) is correct for production
  - Android build artifacts are up-to-date (`android/` directory sync status)

### 1.2 Mobile UX & Responsiveness

- **Files:** `src/components/Layout.tsx`, `src/components/Sidebar.tsx`, `src/components/ui/MobileModal.tsx`, `src/components/ui/MobilePrimitives.tsx`, all `src/pages/*.tsx`
- **Check:**
  - Bottom nav thumb-zone ergonomics (`env(safe-area-inset-bottom)` usage)
  - Touch target sizes (min 44×44dp equivalent)
  - Modal sheet behavior on small screens
  - Form input UX on mobile (numeric keypads, autocomplete, autocorrect)
  - Scroll locking and body overflow handling
  - Dark mode consistency across all mobile components

### 1.3 Mobile Performance

- **Files:** `src/context/ErpContext.tsx`, `src/pages/Dashboard.tsx`, `src/pages/Dispatch.tsx`
- **Check:**
  - Large list rendering (virtualization needed?)
  - PDF generation blocking main thread on mobile
  - Image handling (camera capture, base64 vs file URIs)
  - localStorage backup size limits (~5MB on WebView)
  - Sync queue memory pressure
  - Bundle size and code-splitting effectiveness

### 1.4 Platform-Specific Bugs

- **Files:** `src/lib/biometrics.ts`, `src/lib/nfc.ts`, `src/lib/barcode.ts`, `src/lib/camera.ts`, `src/lib/escpos.ts`
- **Check:**
  - Biometric auth edge cases (cancel, fallback, credential expiry)
  - NFC scan timeout and cleanup
  - Barcode scanner module availability check
  - Camera permission denied handling
  - Bluetooth printer reconnection logic

## Phase 4: Bug Hunt & Logic Gaps

### 4.1 Known Suspect Areas

- **Invoice cancellation** (`src/pages/Invoices.tsx`): slip unlinking uses `null as unknown as undefined` — type unsafe and may cause sync issues
- **Slip status workflow**: No validation preventing illegal transitions (e.g., Cancelled → Pending)
- **Customer balance**: `getCustomerBalance` ignores `amountPaid` on slips — potential discrepancy
- **Employee soft-delete**: `deleteEmployee` doesn't check for active salary transactions
- **Vehicle auto-save**: `CreateSlipForm` creates vehicles silently — may create duplicates on rapid entry
- **Combobox mobile**: Custom combobox may have focus trapping issues on iOS Safari

### 4.2 Error Handling

- **Files:** `src/components/ErrorBoundary.tsx`, all async handlers
- **Check:**
  - Unhandled Promise rejections
  - Network error user messaging
  - Fallback UI completeness
